// the pain

Fragmentation has a
tax.

  • Every phase rebuilds the same contract: spec, mock, fixture, test stub, CI seed
  • Every handoff drops information — field renames, enum shifts, format drift
  • Spec changes? Redo the rebuild in five places. Welcome to fixture rot.
  • Your AI sees one phase at a time. It can't reason across the lifecycle.
  • The verification gap isn't speed — it's the missing connective tissue

// phase 2 / week 1

Backend ships.
Capture once.

Without MockKit

  • Frontend devs hand-update fixtures when API changes
  • Spec drift discovered in QA — weeks late
  • JWTs and timestamps in fixtures expire / break replays
  • "Works on my machine" because everyone has different fixtures

With MockKit

  • One mockkit record command captures real responses
  • Spec drift caught at capture — immediate signal
  • --templatize makes recordings reusable across time
  • One file, committed to git — everyone has the same fixture

// phase 4 / pre-merge

Tests
write themselves.

DEV
Record a Playwright test for: click Create Incident, fill Title with "DB outage", set severity high, submit, verify it shows in the list
AI
[activates /record-flow-as-test]
→ starts MockKit on api.yaml (seed 42)
→ drives app via Playwright MCP
→ captures aria snapshots per step
→ synthesizes tests/create-incident.spec.ts (18 lines, self-cleaning)
→ runs once: passed in 1.4s

Self-cleaning = the generated test deletes the entity it created in afterEach. No suite pollution. No flaky parallelism.

Circuit Breaker

Per-endpoint failure thresholds + cooldowns. Test fallback paths under realistic resilience patterns.

Real result: enterprise demo suite went 13s → 1.9s with audit-tests, all green. Not by deleting tests — by removing test theater.

// get started

Three commands.
Whole lifecycle.

# 1. install
npm install -g @up/mockkit

# 2. add MCP server (one-time per AI client)
echo '{"mcpServers":{"mockkit":{"command":"npx","args":["@up/mockkit-mcp"]}}}' > .mcp.json

# 3. clone the skills (or wait for npm package)
git clone github.com/pallavL01/mockkit && cp -r mockkit/.agents .

Then say: "Stand up a mock for ./api.yaml and build the IncidentList component end-to-end."
